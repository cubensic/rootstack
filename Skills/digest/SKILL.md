# Skill: digest
*Weekly synthesis of journal entries and sessions → `me.md` Current Focus, patterns, lessons, and background.*

---

## When to Run

- Weekly (checked by maintenance-check.py)
- Manually: "Run my digest", "Update me.md", "Synthesize my journal"

---

## What This Skill Does

Reads recent journal entries and AI session logs and produces up to four outputs, each on its own cadence:

**Weekly outputs:**
1. **Updates the `## Current Focus` section of `me.md`** — top 1–3 priorities right now, one line each. This is the only section of `me.md` that digest touches. Everything else in `me.md` is stable identity and should not be modified here.
2. **Appends to `About [You]/patterns/lessons-learned.md`** — if a clear lesson emerged from recent experience. A single strong session can produce a lesson.
3. **Appends to `About [You]/background.md`** — if identity-relevant material surfaced (rare, but checked weekly).

**Monthly output:**
4. **Appends to `About [You]/patterns/behavioral-patterns.md`** — only if a repeating behavior is supported by 3+ evidence points across different entries. Patterns need time to emerge — this looks at the full 30-day window, not just the last week.

**Rule:** Always propose changes and ask for approval before writing anything. Never silently overwrite.

---

## Step-by-Step Instructions

### Step 0 — Harvest sessions (if not already done)

Before reading journals, run the session-harvest skill to make sure recent AI sessions have been captured and summarized:

1. Run `python Skills/session-harvest/harvest.py` to copy raw session files
2. Check for unprocessed sessions in `Knowledge Base/Raw/Sessions/` and summarize them into `Knowledge Base/Sessions/`
3. Follow the full session-harvest SKILL.md instructions for this step

If session-harvest has already been run recently (summaries exist for the current period), skip this step.

---

### Step 1 — Read the journals and session summaries

Read from two time windows:

**Last 7 days (for weekly outputs — me.md Current Focus, lessons, background):**
- `Journal/Personal/` — files dated within the last 7 days (filenames are `DD-MM-YYYY.md`)
- `Journal/Work/` — same
- `Knowledge Base/Sessions/` — session summaries from the last 7 days (filenames are `YYYY-MM-DD [topic].md`)

**Last 30 days (for monthly output — behavioral patterns):**
- Same three folders, but the full 30-day window. Patterns need more data to be reliable.

Session summaries are generated by the session-harvest skill from raw Claude Code transcripts. They contain what was built, decisions made, things learned, and what was hard — often the richest source of signal for patterns and lessons.

If all three folders are empty or contain no recent entries, stop and tell the user: "No journal entries or session logs found from the last 7 days. Add some entries and run this again."

Also read for context:
- `me.md` — current identity + current focus (to understand what's changing, and to preserve the rest of the file)
- `About [You]/goals/1-year.md` — annual goals (for context on priorities)

---

### Step 2 — Synthesize

Identify the following across all journal entries read:

**For `me.md` Current Focus:**
- What are the 1–3 things the person spent the most time on or mentioned most often?
- Rank by weight. The top priority goes first.
- Each priority is **one line** — a crisp noun phrase or short sentence. Not a paragraph.
- This section is small on purpose. If you're tempted to add context, resist — it belongs in `now.md` (which was retired) or the journal. `me.md` is the tight always-loaded version.

**For behavioral patterns (monthly — use the full 30-day window):**
- Did any behavior repeat 3+ times across separate entries over the past 30 days?
- Is it worth naming as a pattern (i.e., does it have consequence — positive or negative)?
- If yes, draft a new pattern entry using the template in the file.
- If a pattern already exists in the file and this data strengthens it, note that instead.
- If the last 30 days don't have enough data for a confident pattern, skip this output entirely. Don't force patterns from thin evidence.

**For lessons learned (only if a clear lesson emerged):**
- Was there a distinct moment of learning — a mistake, a realization, a surprise outcome?
- Can it be stated as one clear sentence the person now knows?
- If yes, draft a new lesson entry using the template in the file.

**For background (only if identity-relevant material appears):**
- Did a journal entry or session reveal a formative experience, turning point, or origin story detail not already captured in `About [You]/background.md`?
- Read `background.md` first to avoid duplicating what's already there.
- If new material exists, propose appending it to the appropriate section (Turning Points, Identity Threads, or Current Chapter).
- This is how background.md evolves over time without requiring the user to sit down and write it all at once.

---

### Step 3 — Propose changes

Present your proposed updates to the user before writing anything. Use this structure:

```
## Digest — [date range covered]

### me.md Current Focus — Proposed Update
Replacing the current Current Focus section with:

1. [Priority 1 — one line]
2. [Priority 2 — one line]
3. [Priority 3 — one line]

---

### Behavioral Patterns — [Proposed new entry / No new entry]
[Draft entry, or: "No new pattern met the 3-evidence threshold."]

---

### Lessons Learned — [Proposed new entry / No new entry]
[Draft entry, or: "No distinct lesson extracted."]

---

### Background — [Proposed addition / No new material]
[Draft addition, or: "No new identity-relevant material found."]

---

Ready to write? Reply "yes" to apply all, or tell me which parts to skip or change.
```

---

### Step 4 — Write on approval

Once the user approves (fully or partially):

- **me.md Current Focus section:** Replace only the content inside the `## Current Focus` section. Do not touch any other section of `me.md`. Update `*Last updated:*` at the top.
- **behavioral-patterns.md:** Insert the new entry below the opening comment block (`<!-- TEMPLATE ... -->`), above the closing `<!-- Add new patterns above this line -->` marker. Update `*Last updated:*` at the top.
- **lessons-learned.md:** Insert the new entry below the opening comment block, above the `<!-- Add new lessons above this line -->` marker. Update `*Last updated:*` at the top.
- **background.md:** Append the new material to the relevant section. Update `*Last updated:*` at the top. Do not rewrite existing content — only add.

Never delete existing pattern, lesson, or background entries. Only append.

---

## Output Format Reference

### me.md Current Focus section
```
## Current Focus

*Top 1–3 priorities right now. Updated weekly by `digest`.*

1. [Priority 1 — one line]
2. [Priority 2 — one line]
3. [Priority 3 — one line]
```

Only 1–3 bullets. No paragraphs, no nested detail. The rest of the dashboard context (active projects, state of mind, challenges, open questions) that used to live in `now.md` should surface through journal entries, not here.

### behavioral-patterns.md entry
```
## [Pattern Name]

**Pattern:** [One sentence]

**Evidence:** [Journal dates and brief descriptions]

**Why it matters:** [Consequence — what it costs or creates]

*Last noted: [date]*

---
```

### lessons-learned.md entry
```
## [Lesson title] — [date]

**The lesson:** [One clear sentence]

**Context:** [What led to this insight]

**How to apply it:** [What changes in behavior or approach]

---
```
