# Projects
*Active projects, businesses, and initiatives. One subfolder per project.*

## Structure

Each project lives in its own subfolder:

```
Projects/
  [Project Name]/
    README.md        ← What it is, current status, quick navigation
    [other files]    ← SOPs, playbooks, notes, strategy docs
```

## Project README Template

Each project's README.md should cover:
- **What it is** — one sentence description
- **Current status** — active / paused / completed
- **Goal** — what success looks like
- **Key files** — navigation guide for the project folder
- **Next action** — what needs to happen next

## Tips

Keep project folders lean. If a document hasn't been touched in 3+ months and the project is still active, it's probably stale — either update it or archive it.

Use the `(Template) Project.md` file in this folder to start a new project quickly.
