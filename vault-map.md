## `Knowledge Base/` — Three-Layer Wiki

Follows the Karpathy three-layer pattern: **Raw sources → Wiki pages → Schema**. A single ingested source typically creates or updates 5–15 wiki pages through cross-referencing.

| Path | Purpose |
|---|---|
| `index.md` | Master catalog of all KB content — AI reads this first to navigate |
| `log.md` | Append-only chronological record of all KB operations |
| `Notes/work/`, `Notes/personal/`, `Notes/my-content/` | Wiki pages — LLM-maintained summaries, entity pages, concept pages, cross-references |
| `Raw/Sources/` | **Immutable** snapshots of ingested articles/papers/threads. Never modify after ingest. |
| `Raw/Sessions/` | Raw `.jsonl` Claude Code transcripts. Never edit. |
| `Sessions/` | Processed session summaries (markdown) — input for `digest` |

**Ingest:** Say "ingest this" + URL/text. **Query:** Ask naturally — AI reads `index.md`. **Lint:** "lint my KB".

## `Projects/` — Active Work

One subfolder per project. Each project folder has its own `README.md` covering: what it is, current status, goal, key files, next action. Use `(Template) Project.md` to scaffold a new project.

## `Skills/` — AI Automation

Reusable AI workflows as markdown files. See `Skills/index.md` for the full catalog with triggers and cadences.

Two categories: **setup skills** (self-delete when complete) and **core skills** (permanent, scheduled or on-demand).

---

## File Conventions

- **Markdown only** for user-facing content
- **Dated files:** `YYYY-MM-DD [title].md` for KB sources and processed sessions; `DD-MM-YYYY.md` for journals and content schedule
- **Template files:** `(Template) [Type].md` — copy these to scaffold new files, don't edit in place
- **Append-only:** `patterns/`, `lessons-learned.md`, `log.md` — never delete existing entries
- **Immutable:** anything in `Knowledge Base/Raw/` — read-only after creation
