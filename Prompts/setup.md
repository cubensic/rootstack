# First-Time Setup Prompt
*Paste this into your AI tool (Claude Code, Codex, Cursor, etc.) the first time you open your Rootstack vault. The AI will guide you through filling it in.*

---

```
You are helping me set up my Rootstack vault for the first time.

Rootstack is a personal context layer — a folder of markdown files that gives you a complete, portable model of who I am so you can work with me more effectively across sessions and tools.

Start by reading these files to understand the system:
- README.md (vault structure and how to navigate it)
- GETTING_STARTED.md (setup overview)
- PRD.md (full product specification — read sections 7.1 and 7.2 carefully)

Then guide me through filling in the vault, one section at a time, in this order:

**Step 1 — About [You]: Foundation files**
These are the stable core of who I am. For each file, ask me questions conversationally, then draft the file from my answers and show it to me for approval before writing.

- About [You]/background.md — my origin story, career history, and current chapter
- About [You]/values.md — my core values, what drives my decisions, non-negotiables
- About [You]/personality.md — how I think, what energizes me, how I communicate

**Step 2 — Content voice**
- Content/Style/writing-style.md — my voice, tone, and style rules

For this file, ask me to paste 2–3 examples of my best writing first. Analyze them before asking questions.

**Step 3 — Goals**
- About [You]/goals/1-year.md — this year's targets
- About [You]/goals/5-year.md — medium-range vision

**Step 4 — Working style**
- About [You]/operating-manual.md — how I work best, how to support me effectively

**Step 5 — now.md**
- About [You]/now.md — current focus, active projects, challenges, state of mind

For this one, ask me what's on my mind right now. This file gets rewritten monthly by the digest skill, so it just needs to reflect the current moment.

---

Rules for this session:
- Do one section at a time. Don't move to the next until I've approved the current one.
- Ask questions conversationally — not as a form. Build on my answers.
- Draft each file using the template structure already in the file (read the file first).
- Show me the full draft before writing. Ask: "Ready to write this?" 
- Write only after I say yes or give edits.
- If I want to stop and resume later, tell me exactly where to pick up: "Next: [file name]."
- Keep the tone of each file consistent with how I answered — don't over-formalize my voice.

Start by reading the files listed above, then introduce yourself and tell me what we're going to do. Then begin with Step 1.
```
